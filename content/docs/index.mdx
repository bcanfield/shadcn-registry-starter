Welcome to the **Shadcn Registry Starter**! This repository serves as a template for building a ShadCN component registry, allowing individuals and organizations to easily share and display custom components, themes, utilities, and hooks. It also functions as a documentation site to guide users through installation and usage.

> **Note:** This repository is **forked from the official [ShadCN UI](https://github.com/shadcn/ui)**. Many modifications have been made, primarily to strip it down to only provide the **registry** and **documentation** portions.

## Key Features

- **Component Registry**: Define and publish your custom ShadCN components, themes, examples, and hooks.
- **Documentation Site**: Provides an organized structure for documentation using MDX files.
- **Integration with ShadCN CLI**: Easily install components into developer environments.

## Registry Overview

The **ShadCN Component Registry** allows users to download custom components, themes, examples, and hooks using the **ShadCN CLI**. Each registry item is defined by its name, type, dependencies, and the files that make up the registry item.

<Tabs defaultValue="item-types">
  <TabsList>
    <TabsTrigger value="item-types">Item Types</TabsTrigger>
    <TabsTrigger value="file-structure">File Structure</TabsTrigger>
  </TabsList>

  <TabsContent value="item-types">
    - **Examples**: Pre-built code snippets.
    - **UI Components**: Ready-to-use user interface elements.
    - **Themes**: Custom styling themes.
    - **Hooks**: Useful React hooks for your project.
  </TabsContent>

  <TabsContent value="file-structure">
    <Steps>
      <Step>
        The `registry` folder contains the following files:
      </Step>
      <Step>
        <pre><code>
        registry/
        ├── registry-examples.ts
        ├── registry-hooks.ts
        ├── registry-themes.ts
        ├── registry-ui.ts
        └── (subfolders for code: examples, hooks, ui, etc.)
        </code></pre>
      </Step>
      <Step>Each file defines arrays representing individual registry items. The structure for each object includes:</Step>
      <ul>
        <li><strong>name</strong>: The item name used for downloading.</li>
        <li><strong>type</strong>: The type of item (example, theme, hook, etc.).</li>
        <li><strong>dependencies</strong>: NPM packages that the item depends on.</li>
        <li><strong>registryDependencies</strong>: ShadCN components required from the official ShadCN registry.</li>
        <li><strong>files</strong>: The code files that make up this registry item.</li>
      </ul>
    </Steps>
  </TabsContent>
</Tabs>

## Building the Registry

Use the following steps to build the registry.

<Steps>
  <Step>Run the build command:</Step>
  <pre><code>npm run build:registry</code></pre>
  
  <Step>
    This script will process the registry files (`registry-examples.ts`, `registry-hooks.ts`, etc.), transforming each entry into a JSON file that meets the ShadCN registry schema. These JSON files are placed into the `public/r/` directory.
  </Step>
</Steps>

## Documentation Overview

All documentation is written in **MDX** and located in the `content/docs/` folder. To build the documentation:

<Steps>
  <Step>Run the build command for docs:</Step>
  <pre><code>npm run build:docs</code></pre>

  <Step>
    The `build:docs` command processes MDX files using Contentlayer, generating type-safe output that can be dynamically served via the documentation site.
  </Step>
</Steps>
